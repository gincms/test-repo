Git in

Gin is a Git-integrated CMS. It keeps everything under version-control, meaning that you can rewind to any moment in the existence of your content — or configuration — at will. It's like being able to travel through time. When using Gin, you basically have superpowers. It also works almost entirely offline.

Awesome.

Get started

Run Gin

The first time you download Gin you'll need to perform some preliminary actions:

  1. Open a terminal
  2. cd to Gin's directory
  3. Run npm install
  4. Run npm start

Once this is done, whenever you want to start Gin, perform steps 1, 2 and 4 (i.e. skip step 3).

NOTE: This process will be simplified in the future to a one-click installation and launch.

Login

To start using Gin, you need to go to the CMS and login via GithHub. If you don't have a GitHub account, you'll need to create one and verify your email address. Then you can revisit the Gin login page and login as normal.

Projects

Overview

When working on content in Gin, you work within the context of projects. Projects are a representation of the underlying Git repository (the technology that makes Gin possible) and contain all your content, as well as all your project-specific settings for Gin CMS (e.g. site name, URL, theme &c).

When you first login you'll not have any of your own projects setup, so the Welcome project is activated by default. The Welcome project is a local and editable version of this website. All of the help pages on this website will be available to you—offline. However, you will not be able to push changes you make locally to the live Gin CMS site.

Create a project

Once logged in, you'll be given the option to create a new project, but it's a good idea to explore the Welcome project to learn what you can do with Gin. Any time other than your first login, when you'd like to create a project:

  1. Go to Settings
  2. Below the Project field, click on Switch project
  3. Click the Create a new project button

Syncing

All changes in Gin are tracked locally within the context of your project. When you're ready to save your changes to the remote repository, you sync the project. Syncing pulls the latest changes from the remote repository, merges your changes into them, and pushes the whole lot back to the remote repository.

This means that almost all work in Gin can be done offline. You only require an internet connection for three actions: login (which only has to be done once), creating projects, and syncing.

To sync your project:

  1. Make sure you have an internet connection
  2. Click Sync in the main menu

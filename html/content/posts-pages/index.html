Posts and pages

The default content type in Gin is the Page. It can be used for both static pages and blog posts. To create a new page:

  1. Click Content Types
  2. Click Add page under the Page

Alternatively, from the Page list, you can click the Add a page button.

Page input form

The input form it pretty straightforward if you've used a CMS before, but there are a couple of few things to know, so below are all the fields you'll find on the edit page, along with a brief explanation of what they're for and how they work.

Title

Required: Put the main title of your content here. A URL is automatically generated which is derived partly from the title field.

Subtitle

Optional: You can add a subtitle to your content.

Slug

Required: This is the location of the content you're editing, relative to the root of your website. When you enter a title, the slug is automatically populated, but you can update it to anything you like (within reason), as long as the constructed URL isn't already in use. Defaults to the URL-friendly version of the title.

Site section

Optional: This field allows you to choose where on your site the currently edited content will live. The site section is the other factor contributing to the final constructed URL. If this field is left blank, the content will live directly off the root of your website (e.g. www.mysite.com/this-is-my-content/). If you fill in the field, your content will live within the path you've added (e.g. www.mysite.com/my-special-section/this-is-my-content/).

Constructed URL

Read-only: This field shows you the URL your content will use once it is published. It is automatically generated and cannot be edited. The only way to change the constructed URL is to change the slug and site section, as outlined above. The link just below this field takes you to a preview of the content that is renedered on-the-fly. The link only appears on content that has been saved at least once.

Body

Optional: Enter you main page/post content in this field. Even though this field is optional, you'll obviously see an empty page if you don't fill it with your content. It's only optional for convenience, in case for some reason you want to add a stub for a post/page but, for whatever reason, will only add the content later.

Writing content in Gin is done using Markdown. The editor has various useful features—including some enhancements from Gin—that help to speed up the writing process. There's a dedicated page to learn more about the editor's features.

Featured image

Optional: Although images can be embedded in your content, sometimes (or maybe all the time) you want a main image that will be displayed at the top of the post, in lists, and on social media channels (Twitter, Facebook, &c). If this is the case, add a featured image.

Summary

Optional: In some cases you want a customised summary/overview of your content rather than a truncated version of your first paragraph. This is particularly useful for content lists, as well as for sharing on social media channels, and for improving your metadata for search.

Blog

Optional: Indicates whether or not the content should be treated as a blog post or (if unchecked) a static page. How static pages are displayed will depend on the theme, but the convention is for static pages to not be included general post lists and they won't display published dates or authors. Defaults to checked—treat as a blog post.

Created

Read-only: When the content was first created.

Updated

Read-only: When the content was last updated.

Published

Read-only: When the content was first published.
